EXPRESS IN A NUTSHELL!

Introduction

Express is simply a web application framework for Node Js.

In every Node js application there is always a package.json file which contains the project's metadata including packages that depend on it to run.Below is an example of package.json file.


{
    "name": "application-name",
    "version": "0.0.0",
    "private": true,
    "scripts": {
        "start": "node ./bin/www"
    },
    "dependencies": {
        "express": "~4.9.0",
        "body-parser": "~1.8.1",
        "cookie-parser": "~1.3.3",
        "morgan": "~1.3.0",
        "serve-favicon": "~2.1.3",
        "debug": "~2.0.0",
        "jade": "~1.6.0"
    }
}

The Ingredients

To create an Express project you will need at least the following key ingredients:

  1. Node and npm
  2. Express generator
  3. Git
  4. Terminal/Command-line interface
  5. Heroku

Let's Cook!!

To install Node and npm visit Node js Official Website

Next install express globally by running the command below:

npm install -g express-generator

To check the express version simply execute : 

express --version

If all is well then go ahead and create a folder and name it hello_express through your terminal

mkdir hello_express

Then navigate to your folder

cd hello_express

Now run the command below to create an express project

express

This will create a bunch of folders and files inside the hello_express directory.Next you will need to install dependancies.This is simply done by running:

npm install

Let's give our application a test drive to see how far we've come with a few lines of code.Simply run:

npm start

Head on over to localhost:3000 and voila!

Nodemon

A Node application compiles before running, so if you make changes to the application code while it’s running, they won’t be picked up until the Node process is stopped and restarted.This is where Nodemon comes in. It automatically restarts the application in case of any changes. To install nodemon globally stop the Node process by pressing CTRL + C then run:

 sudo npm install -g nodemon

Now restart the application by running 

nodemon

Modifying Express for MVC

MVC stands for Model-View-Controller and it aims at seperating our structure into data(model), the display(view) and the application logic(controller). We are simply making code mre maintainable and reusable.A request comes into the application. The request gets routed to a controller. The controller, if necessary, makes a request to the model. The model responds to the controller. The controller sends a response to a view. The view sends a response to the original requester.

Rearrange Things

To keep everything tidy create a folder called app_server and inside it create two folders namely models and controllers. Move the views and routes folders into the app_server folder. If you try to run the application right now it won't work.We need to tell express that we have added new folders and moved routes and views. Head on over to app.js file and change 

app.set('views', path.join(__dirname, 'views'));
to 
app.set('views', path.join(__dirname, 'app_server', 'views'));

Let's also tell express that we have moved the routes.In app.js change 

var routes = require('./routes/index');
var users = require('./routes/users');
to 
var routes = require('./app_server/routes/index');
var users = require('./app_server/routes/users');

Taking the controller code out of the route.

Head on over to app_server/route/index.js file and edit it: 


var homepageController = function (req, res) {
    res.render('index', { title: 'Express' });
};
/* GET home page. */
router.get('/', homepageController);

Setting up the Homepage Controller

For this we will need to create a main.js file inside app_server/controllers folder with the following code. 


/* GET home page */
module.exports.index = function(req, res){
res.render('index', { title: 'Express' });
};

This is how the index.js file in app_server/routes folder should finally look like.


var express = require('express');
var router = express.Router();
var ctrlMain = require('../controllers/main');
/* GET home page. */
router.get('/', ctrlMain.index);
module.exports = router;

Let's go the extra mile and add some Bootsrap to our app. We notice at this point that our views are in .jade format rather than the normal HTML we are all used to. No need to worry.

In our layout.jade file add the following link at the top 


meta(name='viewport', content='width=device-width, initial-scale=1.0')
link(rel='stylesheet', href='https://bootswatch.com/2/superhero/bootstrap.min.css')'

Add jQuery and boostrap.js links at the bottom like this 


script(src='http://code.jquery.com/jquery-1.12.4.min.js')
script(src='https://cdnjs.cloudflare.com/ajax/libs/twitter-bootstrap/4.0.0/js/bootstrap.min.js')

Finally delete the contents of the style.css file to prevent them from overriding the Bootstrap.Verify that your application is running once again.

Making it live on Heroku

There are a few platform as a service providers such as Google Cloud Platform, Nodejitsu, OpenShift, and Heroku.We are going to use Heroku here, but nothing stops you from being adventorous and exploring the other service providers.So lets get started.

Update package.json and add the node and npm versions so that your final package.json file looks like this


{
    "name": "hello_express",
    "version": "0.0.1",
    "private": true,
    "scripts": {
        "start": "node ./bin/www"
    },
    "engines": {
        "node": "~4.2.1",
        "npm": "~2.2.0"
    },
    "dependencies": {
        "express": "~4.9.0",
        "body-parser": "~1.8.1",
        "cookie-parser": "~1.3.3",
        "morgan": "~1.3.0",
        "serve-favicon": "~2.1.3",
        "debug": "~2.0.0",
        "jade": "~1.6.0"
    }
}

Ofcourse you will have to put your own versions in place of mine.

Next create a Procfile at the root of your project. It's sole purpose is to declare the process and commands used to run the application. Add the following line to your Procfile

web: npm start

We are going to push the site to heroku using git. This means that we need to initialize our project as a git repository. Just execute:

git init

Next add everything in the folder to the repository by running this:

git add .

Finally commit changes to the repository

git commit -m "First commit"

The next step would be to create a heroku application. THis would ofcourse mean that you would need to have a heroku account first. Visit Heroku's Official Site and create and account.

Once you are done move swiflty back to the terminal and run 

heroku login

You will be prompted for your heroku credentials. Next create an a heroku application by running the code below on your terminal

heroku create

Deploy the application to heroku by running:

git push heroku master

If successful then your app will be deployed at the provided url. If you log in to heroku's site you will notice that the app has been added.Before you can view the application online you need to add a single web dyno. This is easily done with a simple terminal command:

heroku ps:scale web=1

View application live

heroku open
