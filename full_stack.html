Why full Stack?

Why learn full stack? What is full stack to begin with? Full stack development in layman's language means developing all parts of the web.It starts with the database and web server in the back end, contains application logic and control in the middle, and goes all the way through to the user interface at the front end.

Phew!! After explaining all that it may sound like a whole lot of work. Actually it isn't. It is pretty easy if you take your time to learn it. And perhaps i still haven't answered the big question "Why go full stack?". The greatest benefit of being a full stack developer is the ability to develop apps end-to-end without the dependancy from another developer. You are able to view the full scope of the app as compared to focusing on only one end. If that doesn't sound great to you then i dont know what will.

Why MEAN stack specifically?

The joy of using MEAN stack is that you can code front to back using the same language.It uses Javascript throughout.

Mean Stack pulls together Mongo(M) + Express(E) + Angular(A) + Nodejs(N).

NODE JS

Being the last to be mentioned in the stack does not specifically mean the least. Node.js is a software platform that allows you to create your own web server and build web applications on top of it.

Node.js, when coded correctly, itâ€™s extremely fast and makes very efficient use of system resources.How Does it do this? .Node Js is light on system resources because it is single threaded. Single Threaded?Rather than giving each visitor a unique thread and a separate silo of resources, every visitor joins the same thread. A visitor and thread only interact when needed, when the visitor is requesting something or the thread is responding to a request.

Then there's this other guy - npm.

npm is a package manager that gets installed when you install nodejs. It gives you the ability to download and install Node js modules and 'packages' to extend the functionality of your application.

EXPRESS FRAMEWORK

Express is a web application framework for Node js. THe simple way of describing a framework is a tool that makes work easier. Provides a simpler way of doing things. Express sets up a web server to listen to incoming requests and return relevant responses.Express also provides a really simple interface for directing an incoming URL to a certain piece of code.(Routing urls to reponses) Not to forget that it also provides a number of template engines that make it easier to build HTML pages.
It would be a crime if i did not mention that Express uses sessions so that you can identify individual visitors/users through multiple pages. Imagine if you were to login in each page of an app. Thank you Express!

MongoDB: The Database

MongoDB fits into the satck incredibly well.Just like Node JS, its fast and scalable.MongoDB is a document database. Unlike relational database where the concept of rows and columns are held, MongoDB only keeps the rows but columns do not exist. I read somewhere that Mongo means "HUGE". -That's just a by-the-way. Anyway, back to serious matters.
MongoDB stores documents as BSON which is Binary JSON whereby JSON is Javascript Serialized Object Notation.


{
    "firstName" : "Simon",
    "lastName" : "Holmes",
    _id : ObjectId("52279effc62ca8b0c1000007")
}

N.B:MongoDB IS NOT A TRANSACTIONAL DATABASE! A transactional database can take a number of transactions such that if one fails then the whole transaction fails.Mongo does not work that way. It will take each of the operations independently and if one fails then the transaction will still continue. Imagine if it were an ecommerce site where one fails to submit the payment and the transaction continues to checkout. This would be a fail.

Mongoose adds an entire layer of features on top of Mongo DB that are useful when building web applications. Mongoose makes it easier to manage the connections to your Mongo DB database, as well as to save data and read data.

AngularJS: The front-end framework

Angular js is a javascrip framework for working with data directly on the front end.You are however not to confuse it with jQuery.Angular js and Jquery do not work in the same way. Jquery is a library while Angular is an opinionated framework meaning it forces its opinion on you on how it is to be used.Angular js helps to put the HTML together based on the data provided and updates it if the data changes. Look, jQuery is generally added to a page to provide interactivity after the HTML has been sent to the browser while Angularjs comes a step earlier and helps put the HTML together based on the data provided.

How the MEAN stack components work together

Mongo DB stores data in binary JSON , which through Mongoose is exposed as JSON . The Express framework sits ontop of Node.js, where the code is all written in JavaScript. In the front end is Angular JS , which again is JavaScript.
